# Как собрать и выпустить приложение — полная инструкция

Пошагово, для человека, который никогда этого не делал. Всё нужно делать
на **Mac** (iOS-приложения нельзя собрать на Windows/Linux — этого требует
сама Apple).

## 0. Что должно быть готово заранее

- **Mac** (любой, macOS 13 или новее — лучше свежий).
- **Apple ID**, зарегистрированный в **Apple Developer Program**
  (платная подписка, $99 в год, оформляется на developer.apple.com/programs —
  без неё нельзя загрузить в TestFlight/App Store).
- Стабильный интернет (Xcode и симуляторы весят много гигабайт).

---

## 1. Установи программы на Mac

1. **Xcode** — открой **App Store** на Mac → найди «Xcode» → «Установить».
   Это долго (несколько ГБ), дай ему скачаться полностью.
2. После установки открой Xcode один раз, он предложит доустановить
   доп. компоненты — согласись, подожди.
3. **Homebrew** (менеджер пакетов для Mac) — открой приложение
   **Terminal** (Программы → Утилиты → Терминал) и вставь:
   ```
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
   Нажми Enter, введи пароль от Mac, когда попросит (пароль не будет
   отображаться при вводе — это нормально).
4. **CocoaPods** (нужен для iOS-зависимостей) — в том же Терминале:
   ```
   sudo gem install cocoapods
   ```
5. **Flutter SDK** — в Терминале:
   ```
   cd ~
   git clone https://github.com/flutter/flutter.git -b stable
   echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.zshrc
   source ~/.zshrc
   flutter doctor
   ```
   `flutter doctor` покажет список — если где-то красный крестик (кроме
   Android Studio — она не нужна), пусть Терминал прочитает, что там
   написано, там обычно понятная подсказка что доустановить.

## 3. Подготовь проект к запуску

Находясь в папке `arash-courier` в Терминале:

```

cd ios
pod install
cd ..
```

`pod install` может занять несколько минут — это нормально.

## 4. Первый пробный запуск (проверить, что всё работает)

1. В Терминале, из папки `arash-courier`, набери:
   ```
   open ios/Runner.xcworkspace
   ```
   Откроется Xcode. **Важно**: всегда открывай именно `.xcworkspace`,
   а не `.xcodeproj` — иначе будут ошибки сборки.
2. Слева вверху в Xcode, рядом с кнопкой ▶ (Play), выбери устройство —
   либо подключённый по USB iPhone (разблокируй его, при первом
   подключении на телефоне появится запрос «Доверять этому компьютеру?» —
   подтверди), либо любой симулятор из списка (например «iPhone 16»).
3. Один раз нужно настроить подпись (Signing):
   - Слева в списке файлов нажми на самый верхний пункт **Runner** (с
     синей иконкой).
   - В центре экрана выбери вкладку **Signing & Capabilities**.
   - И для таргета **Runner**, и (если уже добавлен) для **ShareExtension**
     в поле **Team** выбери свой Apple ID/команду разработчика из списка
     (если аккаунта нет в списке: Xcode → Settings → Accounts → «+» →
     войди со своим Apple ID).
   - Убедись, что стоит галочка **Automatically manage signing**.
4. Нажми ▶ (Play) в левом верхнем углу Xcode. Первая сборка идёт долго
   (5-15 минут) — жди, не закрывай Xcode.
5. Приложение должно установиться и открыться — на симуляторе само, на
   реальном iPhone может попросить в Настройках телефона (Основные →
   VPN и управление устройством) подтвердить доверие разработчику
   один раз.
6. Пройдись по приложению: вход, список заказов, создание заказа
   (папка → «+» у менеджера), фото/видео вложения, поиск, скан
   штрих-кода. Если что-то не работает — присылай мне скриншот/описание,
   разберёмся.

## 6. Подготовка к загрузке в TestFlight

1. Перед КАЖДОЙ новой сборкой для TestFlight нужно немного повысить номер
   версии, иначе Apple не даст загрузить (номер уже был использован).
   Открой файл `pubspec.yaml` в корне проекта (можно прямо в Xcode или в
   любом текстовом редакторе), найди строку:
   ```
   version: 1.0.3+4
   ```
   Это «1.0.3» — видимая версия, «+4» — номер сборки. Для каждой новой
   загрузки увеличивай **число после плюса** минимум на 1, например
   `1.0.3+5`, потом `1.0.3+6` и так далее. Видимую версию (1.0.3) меняешь
   только когда делаешь по-настоящему новый релиз для пользователей.

2. Убедись, что в Xcode выбрано устройство **Any iOS Device (arm64)**
   (не симулятор!) — вверху, там же где раньше выбирал iPhone/симулятор,
   в списке будет пункт "Any iOS Device".

## 7. Archive и загрузка в TestFlight

1. В строке меню Xcode: **Product → Archive**. Подожди, идёт сборка
   (может занять 10-20 минут).
2. Когда сборка закончится, откроется окно **Organizer** со списком
   архивов. Выбери свежий (сверху) → нажми справа кнопку
   **Distribute App**.
3. Выбери **App Store Connect** → Next.
4. Выбери **Upload** → Next.
5. Оставь галочки по умолчанию (обычно "Automatically manage signing" уже
   отмечено) → Next → Next → **Upload**.
6. Дождись надписи «Upload Successful». Дальше обработка на серверах
   Apple — может занять от 15 минут до пары часов, придёт письмо на
   почту, когда сборка обработается.

## 8. Тестирование в TestFlight

1. Зайди на [appstoreconnect.apple.com](https://appstoreconnect.apple.com) →
   свой аккаунт → **My Apps** → выбери приложение Arash Courier.
2. Вкладка **TestFlight** сверху.
3. Когда сборка появится в списке (колонка «Processing» сменится на
   готовую) — нажми на неё, укажи «What to Test» (можно просто коротко
   написать, что изменилось, по-русски).
4. Добавь себя/тестеров:
   - **Internal Testing** — самый быстрый способ: добавь людей, у которых
     есть роль в твоей команде App Store Connect (Users and Access) — им
     сборка станет доступна почти сразу, без проверки Apple.
   - **External Testing** — если нужны тестеры без доступа к твоему
     аккаунту разработчика, но тогда первая сборка проходит короткую
     проверку Apple (обычно до суток).
5. Тестер должен установить приложение **TestFlight** из App Store, принять
   приглашение (придёт письмо/ссылка), и оттуда установить Arash Courier.
6. Проверьте на реальных телефонах все функции из этого чата: создание
   заказа менеджером, фото/видео, поиск, «Поделиться в Arash» (если уже
   настроил Share Extension).

---

## 9. Публикация в App Store (только после того, как в TestFlight всё ОК)

1. В App Store Connect → твоё приложение → вкладка **App Store** (не
   TestFlight) сверху.
2. Если версия ещё не создана — нажми **+ Version or Platform** → iOS →
   впиши номер версии (тот же, что в pubspec.yaml, например 1.0.3).
3. Заполни обязательные поля, если ещё не заполнены: скриншоты (для
   нужных размеров экрана), описание, ключевые слова, ссылка на политику
   конфиденциальности, категория приложения, возрастной рейтинг.
4. В разделе **Build** нажми **+** и выбери ту самую сборку, которую уже
   проверили в TestFlight.
5. Внизу справа — **Save**, затем **Submit for Review**.
6. Ответь на пару стандартных вопросов Apple (про экспортное шифрование —
   обычно «No», если не используешь своё шифрование сверх стандартного
   HTTPS) → подтверди отправку.
7. Статус сменится на **Waiting for Review** → потом **In Review** →
   обычно занимает 24-48 часов. Если Apple что-то отклонит — пришлют
   письмо с причиной, обычно легко поправимо.
8. Как только статус станет **Approved**:
   - если стоит **Automatically release this version** — приложение
     появится в App Store само в течение пары часов;
   - если стоит **Manually release** — зайди в App Store Connect и нажми
     **Release This Version** сам, когда будешь готов.

## Частые проблемы

- **"No account for team"** в Xcode → Xcode → Settings → Accounts → «+» →
  войди Apple ID заново.
- **Ошибка сборки после `git clone`** → почти всегда помогает:
  ```
  cd arash-courier
  flutter clean
  flutter pub get
  cd ios && pod install && cd ..
  ```
- **"Missing Compliance"** при загрузке в TestFlight → в App Store Connect
  отметь, что приложение не использует нестандартное шифрование (обычная
  ситуация для большинства приложений).
- Каждая следующая загрузка в TestFlight = обязательно новый номер сборки
  (шаг 6.1 выше) — иначе Xcode откажется загружать.
